Design and Development Process

Research and Planning

Entity Relationship Diagram

Understanding the Brand:
- Worked closely with Verde And Glow to capture their ethos—prioritizing sustainability without sacrificing style.
- Analyzed their customer base: eco-conscious shoppers who value transparency and quality.

Technical Blueprint:
- Designed an 8-table database schema to handle products, variants, categories, and orders.
- Mapped user flows for both shoppers and admins, ensuring intuitive navigation.
- Planned for scalability—like future subscription models or API integrations.

UX Focus:
- Highlighted sustainability info (carbon impact, materials) on every product page.
- Mobile-first design so browsing feels effortless on any device.

Challenges and Solutions

ActiveStorage Image Hiccups
- Problem: Uploads sometimes failed, leaving broken images.
- Solution: Added client-side validation, fallback placeholders, and proper variant processing.

Pagination + Filters Clash
- Problem: Kaminari lost filtered results when paginating.
- Solution: Restructured queries to preserve filters across pages and built reusable UI components.

Admin Dashboard Quirks
- Problem: ActiveAdmin and Ransack had unexpected conflicts.
- Solution: Customized Ransack configurations and added manual filters for complex searches.

Dynamic Pricing Logic
- Problem: Sale prices needed to display alongside original prices reliably.
- Solution: Added a regular_price column with automated tracking and clear visual cues for discounts.

Final Outcome

Features:
- Full-featured product catalog with filtering, search, and pagination
- Admin dashboard for inventory, pricing, and image management
- Mobile-responsive design with Tailwind CSS
- Sustainability-focused UX (carbon labels, ethical sourcing badges)

Lessons Learned:
- Data modeling is everything—a clean schema saves headaches later.
- Admin tools need love too! Clear documentation prevents confusion.
- Error handling isn’t glamorous, but it’s what makes a product feel polished.
