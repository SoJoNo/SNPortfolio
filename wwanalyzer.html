Winnipeg Weather Analyzer

WWA is a Python app that displays records of Winnipeg weather and temperature in previous years.

Project Overview

The Weather Processing App is a Python-based application developed for academic purposes. The project focuses on scraping historical daily weather data — specifically, the minimum, maximum, and mean temperatures — for Winnipeg from Environment Canada's climate archive. The scraped data is structured into a dictionary format and can be used for further analysis or visualization in later stages of the application. This modular scraper adheres to best practices, encapsulating all scraping logic within a single class (WeatherScraper) using Python’s HTMLParser.

Design and Development Process

Research and Planning

Research began with analyzing the Environment Canada daily climate data website’s structure. The team investigated how historical data is embedded within URLs and examined how the HTML tables are structured across different months and years. It was determined that the temperature data resides in a table with the class table-striped, and that each row of interest includes the historical-data class. Manual inspection of URL behavior and View Page Source helped identify where and how data was stored, as well as how to detect when no more data is available.
The scraper was planned to:

  • Iterate backwards in time month-by-month

  • Automatically stop scraping after encountering 3 consecutive months with no data

  • Parse data from static HTML (no JavaScript execution allowed)

Challenges and Solutions

  • Challenge: Initial versions using HTMLParser failed to locate and parse the right data due to misalignment with the actual HTML structure.
    Solution: Refactored parser logic based on the actual source HTML, including use of column index mapping, scoped <th> tags for days, and safe numeric extraction.
  • Challenge: The website uses a complex layout with multiple nested elements and misleading class names.
    Solution: Isolated the exact table and row classes, then dynamically mapped column indices to temperature values.
  • Challenge: Empty or malformed temperature values (e.g., "M", "—") caused ValueErrors.
    Solution: Added try/except blocks to safely skip non-numeric values.
  • Challenge: Page blocking and inconsistent response headers.
    Solution: Used a custom User-Agent header to mimic a real browser and avoid request denial.

Final Outcome

The result is a fully functional Python module (scrape_weather.py) that can reliably scrape structured historical weather data from Environment Canada for Winnipeg. The WeatherScraper class can be invoked from any script and produces a dictionary of weather records in the required format:

It handles dynamic backtracking, error tolerance, and self-terminates when no more historical data is detected. The class is ready to be integrated into the larger Weather Processing App and lays a strong foundation for further features such as data analysis or visualizations.
